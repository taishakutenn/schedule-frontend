# getFilteredData

Файл: `src/utils/getFilteredData.js`

Функция для фильтрации массива объектов по поисковому запросу.

## Описание
Выполняет полнотекстовый поиск по всем полям объекта, преобразуя значения к строке и приводя к нижнему регистру.

## Сигнатура
```javascript
getFilteredData(data, search)
```

## Параметры
| Параметр | Тип | Описание |
|----------|-----|----------|
| `data` | `Array<Object>` | Массив объектов для фильтрации |
| `search` | `string` | Поисковый запрос |

## Возвращает
`Array<Object>` — отфильтрованный массив объектов, где хотя бы одно поле содержит поисковый запрос.

## Алгоритм работы
1. Если `search` пустой — возвращает исходный массив без изменений
2. Преобразует поисковый запрос к нижнему регистру
3. Для каждого объекта проверяет все значения полей:
   - Преобразует значение к строке через `String(value)`
   - Приводит к нижнему регистру
   - Проверяет вхождение поисковой строки через `includes()`
4. Возвращает объекты, где хотя бы одно поле совпало

## Примеры использования

### Базовое использование
```javascript
import { getFilteredData } from './getFilteredData';

const data = [
  { id: 1, surname: "Иванов", name: "Иван" },
  { id: 2, surname: "Петров", name: "Пётр" }
];

const filtered = getFilteredData(data, "иван");
// Вернёт: [{ id: 1, surname: "Иванов", name: "Иван" }]
```

### Поиск по всем полям
```javascript
const data = [
  { group_name: "ИВТ-21", speciality_code: "09.02.07" },
  { group_name: "ПИ-22", speciality_code: "09.02.08" }
];

getFilteredData(data, "09.02");
// Вернёт оба объекта, так как код специальности есть в каждом
```

### Пустой поиск
```javascript
getFilteredData(data, "");
// Вернёт исходный массив без фильтрации
```

### Поиск с учётом регистра (не чувствителен)
```javascript
getFilteredData(data, "ИВТ");
getFilteredData(data, "ивт");
// Оба вызова вернут одинаковый результат
```

## Особенности
- **Регистронезависимый поиск** — "Иван" и "иван" дадут одинаковый результат
- **Поиск по всем полям** — достаточно совпадения в любом поле объекта
- **Частичное совпадение** — используется `includes()`, а не точное сравнение
- **Безопасное преобразование** — `String(value)` работает с любыми типами данных

## Производительность
- **Временная сложность**: O(n × m), где n — количество объектов, m — количество полей в объекте
- **Пространственная сложность**: O(k), где k — количество совпавших объектов

## Оптимизация для больших объёмов
Для больших массивов данных рекомендуется:
1. Дебаунс поискового запроса
2. Мемоизация результатов
3. Серверная пагинация

```jsx
const DebouncedSearch = () => {
  const [search, setSearch] = useState("");
  const debouncedSearch = useDebounce(search, 300);
  
  const filteredData = useMemo(() => 
    getFilteredData(data, debouncedSearch),
    [data, debouncedSearch]
  );
  
  return <div>{/* ... */}</div>;
};
```
