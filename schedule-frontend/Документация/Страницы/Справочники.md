# Описание

Файл: `src\pages\Handbooks.jsx`

Страница `Handbooks` — универсальная страница для работы со справочниками системы. Поддерживает просмотр, добавление, редактирование и удаление записей для 10 типов справочников: преподаватели, категории преподавателей, специальности, группы, формы оплаты, здания, кабинеты, типы занятий и потоки занятий.

# Используемые компоненты

## Внешние компоненты

- [[InfoBlock]] — блок справочной информации в верхней части страницы
- [[Button]] — кнопки управления (добавить, редактировать, удалить, навигация)
- [[HandbookTable]] — таблица для отображения данных справочников
- [[ModalForm]] — модальное окно для добавления/редактирования записей
- [[ConfirmModal]] — модальное окно подтверждения удаления (двухуровневое)
- [[GroupPlate]] — группа кнопок навигации по справочникам

## Хуки

- `useApiData` — загрузка данных из API
- `usePost` — отправка POST-запросов (создание записей)
- `useUpdate` — отправка PUT-запросов (обновление записей)
- `useDelete` — отправка DELETE-запросов (удаление записей)

## Утилиты

- `tableConfig` — конфигурация API-функций для каждого справочника
- `getFilteredData` — фильтрация данных по поисковому запросу
- `tableIds` — конфигурация первичных ключей для удаления
- `displayFieldConfig` — поля для отображения в сообщении об удалении
- `formConfig` — конфигурация полей формы для каждого справочника

# Переменные

## Состояния (useState)

### handbook

`string | null` — текущий выбранный справочник (`teachers`, `groups`, `streams` и т.д.)

### search

`string` — текущее значение поискового запроса

### refreshTrigger

`number` — счётчик для принудительного обновления данных после CRUD-операций

### selectedRowData

`object | null` — данные выбранной строки таблицы

### isModalOpen

`boolean` — открыто ли модальное окно добавления/редактирования

### isConfirmModalOpen

`boolean` — открыто ли первое модальное окно подтверждения удаления

### isFinalConfirmModalOpen

`boolean` — открыто ли второе (финальное) модальное окно подтверждения удаления

### originalGroupsData

`object` — маппинг оригинальных данных групп для корректного редактирования (сохраняет `group_advisor_id` как ID, а не как имя преподавателя)

## Refs

### tableRef

`useRef(null)` — ссылка на контейнер таблицы для обработки кликов вне таблицы

### controlContainerRef

`useRef(null)` — ссылка на контейнер кнопок управления

## Константы

### headerInfo

Массив объектов для компонента InfoBlock с заголовком "Справочная информация".

### handbookTitles

Объект с человеко-читаемыми названиями справочников для отображения в заголовке.

# Функции

## Обработчики переключения справочников

### handleTeacher / handleTeacherCategory / handleCabinet / handleBuilding / handleSpeciality / handleSessionType / handleGroup / handleStream / handlePaymentForm

Устанавливают соответствующее значение `handbook` и сбрасывают `selectedRowData`.

## CRUD-операции

### handleSaveEdit

```jsx
const handleSaveEdit = async (
  updateDataOrFormData,
  idDataOrIdValues,
  onReset
) => { ... }
```

Универсальная функция обновления записи. Поддерживает два режима:

1. **С изменяемым ключом** — когда `idDataOrIdValues` это объект
2. **Стандартный** — когда `idDataOrIdValues` это массив значений

Отправляет PUT-запрос на `${API_BASE_URL}/${handbook}/update`.

### handleAdd

```jsx
const handleAdd = async (formData, onReset) => { ... }
```

Универсальная функция создания записи. Отправляет POST-запрос на `/${handbook}`.

### handleOpenConfirmDelete

Открывает первое модальное окно подтверждения удаления.

### handleConfirmDelete

Закрывает первое модальное окно и открывает второе (финальное).

### handleFinalConfirmDelete

```jsx
const handleFinalConfirmDelete = async () => { ... }
```

Выполняет фактическое удаление записи:

1. Получает конфигурацию ID из `tableIds[handbook]`
2. Извлекает значения первичных ключей из `selectedRowData`
3. Отправляет DELETE-запрос через `useDelete`
4. Обновляет данные через `refreshTrigger`

## Вспомогательные компоненты

### ControlContainer

Функциональный компонент для кнопок управления и поиска.

**Пропсы:**

- `handbook` — текущий справочник
- `onAdd` — обработчик добавления
- `onEdit` — обработчик редактирования
- `onDelete` — обработчик удаления
- `search` — значение поиска
- `onSearchChange` — обработчик изменения поиска

### CountHandbookRows

Компонент для отображения количества отфильтрованных записей.

**Пропсы:**

- `handbook` — текущий справочник
- `data` — исходные данные
- `search` — поисковый запрос

## Обработчики событий

### handleRowClick

```jsx
const handleRowClick = (rowData) => {
  if (handbook === "groups" && rowData.group_name) {
    const originalData = originalGroupsData[rowData.group_name];
    setSelectedRowData(originalData || rowData);
  } else {
    setSelectedRowData(rowData);
  }
};
```

Обработчик клика по строке таблицы. Для групп использует оригинальные данные (с ID преподавателя вместо имени).

# Хуки

## useEffect — Сохранение оригинальных данных групп

```jsx
useEffect(() => {
  if (
    handbook === "groups" &&
    data &&
    !teachersLoading &&
    !teachersError &&
    teachersData
  ) {
    const originalData = {};
    data.forEach((group) => {
      originalData[group.group_name] = group;
    });
    setOriginalGroupsData(originalData);
  }
}, [handbook, data, teachersLoading, teachersError, teachersData]);
```

Сохраняет оригинальные данные групп для корректного редактирования поля `group_advisor_id`.

## useEffect — Клик вне таблицы

```jsx
useEffect(() => {
  const handleClickOutside = (event) => {
    if (
      tableRef.current &&
      !tableRef.current.contains(event.target) &&
      controlContainerRef.current &&
      !controlContainerRef.current.contains(event.target)
    ) {
      setSelectedRowData(null);
    }
  };

  document.addEventListener("mousedown", handleClickOutside);
  return () => {
    document.removeEventListener("mousedown", handleClickOutside);
  };
});
```

Сбрасывает выделение строки при клике вне таблицы.

    isOpen={isModalOpen}
    onClose={() => { setIsModalOpen(false); setSelectedRowData(null); }}

    onClose={() => setIsConfirmModalOpen(false)}

# Логика работы

## 1. Загрузка данных

Данные загружаются через `useApiData` с использованием конфигурации из `tableConfig`. Для справочника групп дополнительно загружаются преподаватели через `getTeachers`.

## 2. Обработка данных для групп

Для групп данные преобразуются:

- `group_advisor_id` (число) → имя преподавателя (строка) для отображения
- Оригинальные данные сохраняются в `originalGroupsData` для редактирования

## 3. Фильтрация

Данные фильтруются через `getFilteredData` по поисковому запросу.

## 4. CRUD-операции

- **Создание**: `handleAdd` → POST `/${handbook}` → `refreshTrigger++`
- **Чтение**: `useApiData` → `renderContent` → `HandbookTable`
- **Обновление**: `handleSaveEdit` → PUT `/${handbook}/update` → `refreshTrigger++`
- **Удаление**: Двухэтапное подтверждение → `handleFinalConfirmDelete` → DELETE `/${handbook}/{id}` → `refreshTrigger++`

## 5. Двухуровневое подтверждение удаления

1. Первое модальное окно: "Подтвердите удаление" — показывает, что будет удалено
2. Второе модальное окно: "Подтвердите удаление ещё раз." — предупреждение о возможных ошибках

# Особенности реализации

## 1. Обработка первичных ключей

Для справочников с составными ключами (кабинеты, потоки) используется массив полей в `tableIds`.

## 2. Редактирование групп

Проблема: при отображении `group_advisor_id` преобразуется в имя преподавателя, но для обновления нужен ID.

Решение: хранение оригинальных данных в `originalGroupsData` и использование при клике на строку.

## 3. Двухуровневое удаление

Предотвращает случайное удаление записей с предупреждением о возможных последствиях.

## 4. Универсальность

Одна страница для всех справочников с конфигурацией через объекты:

- `tableConfig` — API функции
- `formConfig` — поля формы
- `tableIds` — первичные ключи
- `displayFieldConfig` — поля для отображения
